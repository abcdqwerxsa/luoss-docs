# 数据盘

数据盘是用户自建的持久存储（Longhorn RWO 卷），可作为**开发环境的工作盘**（主目录持久化），也可挂到训练任务中使用，或通过文件浏览器在线查看和下载文件。

<FeatureBadge status="beta" />

## 功能概述

| 功能 | 说明 |
|------|------|
| **创建数据盘** | 指定名称和容量（GiB），平台自动创建持久卷 |
| **作为开发环境工作盘** | 创建[开发环境](/guide/environments)时选为工作盘，承载全量持久化 |
| **绑定训练任务** | 创建训练任务时选择数据盘，自动挂载到 `/mnt/data` |
| **浏览/下载文件** | 在线文件浏览器，支持浏览、下载（单文件和批量打包） |
| **删除数据盘** | 删除后数据将被清除 |
| **容量配额** | 每用户总量上限 2 TiB，数量不限 |

::: tip 什么是数据盘
数据盘是一块独占的持久存储空间。你可以把它理解为一个「移动硬盘」：
- 作为开发环境的**工作盘**，承载环境主目录的全部持久化（pip/apt 安装、家目录、配置等）。
- 创建训练任务时可以挂上它，训练数据或输出模型存到这里。
- 训练/环境结束后通过文件浏览器查看和下载里面的文件。
- 同一时间只能被一个目标（开发环境 / 训练任务）使用，用完解绑后可再次使用。
:::

## 创建数据盘

1. 进入 **数据盘** 页面
2. 点击 **新建数据盘**
3. 填写信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| 名称 | 数据盘显示名称 | `模型转换输出盘` |
| 容量 | 存储容量（GiB） | `100` |

4. 点击 **创建**

::: warning 容量限制
- 每用户数据盘**总容量上限为 2 TiB**（2048 GiB）。
- 创建时页面会显示当前已用容量和剩余可用容量。
- 作为开发环境工作盘使用时支持**在线扩容**（在环境详情中操作）；仅用于训练任务/文件浏览的数据盘创建后容量固定。
:::

## 作为开发环境工作盘

数据盘最重要的用途之一，是作为[开发环境](/guide/environments)的**工作盘**——也就是环境主目录的持久化层。开发环境通过 OverlayFS + chroot 把工作盘当作可写层，让容器内的所有修改（pip/apt 安装、家目录、配置等）都落到工作盘上、重启不丢。

### 怎么挂上去

在**创建开发环境**时，「工作盘」一栏二选一：

| 方式 | 说明 |
|------|------|
| **新建数据盘** | 当场创建一块新数据盘作为工作盘（填写容量与名称） |
| **使用空闲数据盘** | 复用一块已有的空闲数据盘——适合把同一份环境/数据带到新环境，或提前在「数据盘」页面准备好 |

绑定后，该数据盘在列表中会显示为「**开发环境工作盘：\<环境名\>**」。

::: warning 工作盘绑定中的限制
- 工作盘被环境占用期间，**不能**在数据盘页面卸载或删除（按钮禁用）。
- 如需释放工作盘：先**删除对应的开发环境**，平台会自动解绑；工作盘数据**保留**，可再挂到下一个环境。
- 一块数据盘同一时刻只能被一个目标（开发环境 / 训练任务 / 文件浏览）使用（RWO 独占）。
:::

### 工作盘扩容

工作盘支持**在线扩容**：在开发环境详情页执行「扩容存储」即可，Longhorn 会在线放大 PVC，无需重启环境。扩容后的容量同步计入数据盘配额。

## 浏览和下载文件

数据盘创建完成后，可以随时通过文件浏览器查看和下载其中的文件，无需启动训练任务。

### 操作步骤

1. 在数据盘列表中，找到状态为 **空闲** 的数据盘
2. 点击 **浏览文件** 按钮
3. 系统会自动在新标签页中打开文件浏览器界面

::: info 文件浏览器登录
文件浏览器需要登录，凭据如下：
- **用户名**：`admin`
- **密码**：`admin12345678`

该凭据为平台预设的固定账号，所有用户共用。
:::

### 文件浏览器功能

| 操作 | 说明 |
|------|------|
| 浏览目录 | 点击文件夹进入子目录 |
| 下载文件 | 点击文件名直接下载 |
| 批量下载 | 多选文件后点击下载按钮，自动打包为 ZIP |
| 预览 | 支持文本、图片等常见格式的在线预览 |

::: warning 只读模式
文件浏览器当前为**只读模式**，不支持上传、修改或删除文件。如需写入数据，请通过训练任务挂载数据盘后在任务中写入。
:::

### 关闭浏览

- 点击数据盘列表中的 **关闭浏览** 按钮即可关闭文件浏览器会话。
- 浏览会话在 **30 分钟无操作** 后自动关闭。
- 关闭浏览后，该数据盘恢复为空闲状态，可再次绑定到训练任务。

## 绑定训练任务

创建训练任务时，可以选择将一块空闲数据盘绑定到任务中，任务运行期间数据盘将挂载到容器的 `/mnt/data` 目录。

### 通过前端绑定

1. 创建训练任务时，找到 **数据盘（可选）** 卡片
2. 从下拉列表中选择一块空闲数据盘
3. 任务启动后，数据盘自动挂载到 `/mnt/data`

::: warning 使用限制
- 数据盘为独占式存储，**同一时间只能绑定到一个训练任务**。
- 仅支持**单副本**（replicas=1）任务。
- 任务删除后数据盘自动解绑，恢复为空闲状态。
:::

### 通过 ktp 命令行绑定

```bash
ktp submit \
  --name my-job \
  --image pytorch:2.1 \
  --npu 8 \
  --max-runtime 1440 \
  -c "python train.py" \
  --data-volume 3
```

`--data-volume` 参数指定数据盘 ID，可在数据盘页面查看。

### 通过 YAML 配置文件绑定

```yaml
name: my-job
image: pytorch:2.1
npu: 8
max_runtime_minutes: 1440
command: "python train.py"
data_volume: 3  # 数据盘 ID
```

## 在训练任务中使用数据盘

数据盘挂载到 `/mnt/data` 目录后，训练脚本可以直接读写该目录：

```python
# 读取数据盘中的训练数据
data = load_data("/mnt/data/dataset/train.jsonl")

# 训练完成后，将模型保存到数据盘
model.save("/mnt/data/output/model.bin")
```

训练任务结束后（停止或删除），数据盘自动解绑。之前写入的数据保留在数据盘中，可通过文件浏览器查看和下载。

## 删除数据盘

1. 确保数据盘处于 **空闲** 状态（未被训练任务占用）
2. 点击数据盘列表中的 **删除** 按钮
3. 确认删除

::: danger 不可恢复
删除数据盘后，其中的所有数据将被**永久清除**，无法恢复。请确保已下载所需文件后再删除。
:::

## 常见问题

### 数据盘被占用了怎么办？

数据盘被占用时会显示绑定对象，按类型处理：

- 「已绑定训练任务」：删除或停止该训练任务后自动解绑。
- 「开发环境工作盘：\<环境名\>」：需**删除对应的开发环境**后才会解绑（数据保留），不能在数据盘页面直接卸载。

### 浏览时提示「浏览会话暂时不可用」

这通常是文件浏览器 Pod 正在启动中（首次访问需要几秒到十几秒）。请稍等片刻后刷新页面重试。

### 文件浏览器标签页被拦截了

如果浏览器弹窗拦截导致标签页未弹出，数据盘列表中该数据盘行会显示 **打开浏览** 按钮，点击即可手动打开。
