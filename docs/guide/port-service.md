# 端口服务

端口服务允许你将开发环境（或命名空间中其他工作负载）的容器端口通过 Kubernetes NodePort 暴露到局域网，让局域网内的其他设备可以直接访问你的服务。

<FeatureBadge status="stable" />

## 功能概述

| 功能 | 说明 |
|------|------|
| **容器发现** | 自动列出命名空间中的容器，一键选择目标 |
| **自动分配** | NodePort 端口由系统自动分配，无需手动指定 |
| **访问地址** | 自动生成访问 URL，支持一键复制 |
| **配额管理** | 每用户限制端口服务数量，防止资源滥用 |
| **TCP/UDP** | 支持 TCP 和 UDP 两种协议 |

::: tip 配额限制
每个用户最多创建 **2 个**端口服务。如需创建新的，请先删除不需要的服务。
:::

## 使用场景

- 在开发环境中启动 Web 服务，让团队成员通过局域网访问预览
- 暴露推理服务端口，供其他设备调用 API
- 调试时临时开放端口，查看服务运行状态

## 创建端口服务

### 步骤

1. 进入 **端口服务** 页面
2. 点击 **创建端口服务** 按钮
3. 填写配置信息：

| 字段 | 说明 |
|------|------|
| 名称（可选） | 为端口服务起一个友好的名称，方便识别 |
| 目标来源 | 选择容器（推荐）或手动输入标签选择器 |
| 容器 / 选择器 | 选择目标容器，或手动输入 JSON 标签选择器 |
| 容器端口 | 容器内服务监听的端口号（1–65535） |
| 协议 | TCP（默认）或 UDP |

4. 点击 **创建**，系统自动创建 NodePort Service 并分配端口

### 方式一：选择容器（推荐）

在创建表单中，**目标来源** 选择「选择容器」：

1. 系统自动加载你命名空间中的容器列表
2. 从下拉菜单中选择目标容器
3. 输入容器端口和协议
4. 点击创建

系统会自动从选中的 Pod 获取标签并生成 Kubernetes Service Selector，无需手动输入。

### 方式二：手动输入选择器

如果你的工作负载不是通过平台创建的，或者需要精确匹配特定标签，可以选择「手动输入选择器」：

1. **目标来源** 选择「手动输入选择器」
2. 输入 JSON 格式的标签选择器，例如：

```json
{"app": "my-inference-server"}
```

3. 输入容器端口和协议
4. 点击创建

## 端口服务列表

| 字段 | 说明 |
|------|------|
| 名称 | 友好名称或 Service 名称 |
| 标签选择器 | 匹配目标 Pod 的标签 |
| 容器端口 | 容器内监听的端口 |
| NodePort | 系统分配的节点端口 |
| 协议 | TCP / UDP |
| 访问地址 | `http://<节点IP>:<NodePort>` |
| 状态 | 正常 / 创建中 / 异常 |

::: info 自动刷新
列表每 30 秒自动刷新状态。你也可以点击 **刷新** 按钮手动刷新。
:::

## 实战示例：快速共享一个静态文件服务

以下演示如何在开发环境中启动一个 HTTP 文件服务，并通过端口服务暴露到局域网。

### 第一步：进入开发环境终端

通过 SSH 连接到你的开发环境，或使用 [Web 终端](/guide/web-terminal)。

### 第二步：准备文件并启动服务

假设你要共享 `/data/output` 目录中的文件：

```bash
# 进入要共享的目录（可以是任意目录）
cd /data/output

# 启动 HTTP 文件服务器，监听 8080 端口，绑定所有网卡
python3 -m http.server 8080 --bind 0.0.0.0
```

看到类似以下输出说明服务已启动：

```
Serving HTTP on 0.0.0.0 port 8080 (http://0.0.0.0:8080/) ...
```

::: warning 绑定地址
必须使用 `--bind 0.0.0.0`，否则服务只监听 localhost，NodePort 无法转发流量。如果是自定义程序，请确保监听地址为 `0.0.0.0` 而非 `127.0.0.1`。
:::

### 第三步：创建端口服务

1. 打开平台 **端口服务** 页面
2. 点击 **创建端口服务**
3. 填写表单：

| 字段 | 值 |
|------|------|
| 名称 | `文件共享` |
| 目标来源 | 选择容器 |
| 容器 | 选择你的开发环境容器 |
| 容器端口 | `8080` |
| 协议 | TCP |

4. 点击 **创建**

### 第四步：访问服务

创建成功后，列表中会显示 **访问地址**，格式为：

```
http://<节点IP>:<NodePort>
```

例如 `http://192.168.1.100:31234`。

在局域网内任何设备的浏览器中打开这个地址，即可看到文件列表和下载链接。

### 第五步：停止服务

使用完毕后：

1. 在终端中按 `Ctrl+C` 停止 Python HTTP 服务器
2. 在 **端口服务** 页面删除对应的端口服务，释放配额

::: tip 提示
端口服务删除后，局域网内将无法再通过该 NodePort 访问。如果只是临时停止了容器内的服务但保留端口服务，重启服务后即可恢复访问。
:::

## 删除端口服务

1. 在端口服务列表中找到要删除的服务
2. 点击 **删除** 按钮
3. 确认删除

删除后：
- Kubernetes NodePort Service 被移除
- 局域网无法再通过该端口访问
- 释放一个端口服务配额

## 常见问题

### Q: 为什么访问地址无法打开？

A: 请检查以下几点：

1. 容器内的服务是否正在运行（`curl localhost:<端口>` 测试）
2. 服务是否绑定在 `0.0.0.0` 而非 `127.0.0.1`
3. 容器端口是否与创建端口服务时填写的一致
4. 你的设备是否与集群节点在同一局域网内

### Q: 端口服务状态显示"异常"？

A: "异常"表示对应的 Kubernetes Service 不存在。可能是 Service 被手动删除了。建议删除该端口服务并重新创建。

### Q: 可以指定 NodePort 端口号吗？

A: 目前不支持指定端口号，NodePort 由 Kubernetes 自动从配置的端口范围内分配。

### Q: 端口服务重启后 NodePort 会变吗？

A: 不会。端口服务对应一个 Kubernetes Service，只要 Service 存在，NodePort 就不会变化。删除并重新创建才会获得新的端口。

### Q: 支持暴露开发环境以外的容器吗？

A: 支持。只要容器在你的命名空间中运行，端口服务就可以选择并暴露它。你可以通过手动输入标签选择器来匹配任意工作负载。
