# 队列管理

队列管理页面用于管理 Volcano 调度队列。

## 功能概述

| 功能 | 说明 |
|------|------|
| **队列列表** | 查看所有队列 |
| **队列详情** | 查看队列配置 |
| **创建/编辑** | 创建和编辑队列 |
| **删除队列** | 删除不需要的队列 |

## 队列层级关系

系统使用扁平队列模式，所有队列直接挂在 root 下：

```
root
├── debug-sub (调试池)
├── normal-queue (普通用户共享)
├── user-alice-compute
├── user-bob-compute
└── ...
```

### 父子队列约束

::: warning 重要约束
1. **子队列资源不能超过父队列**：子队列的 Deserved 和 Capability 不能超过父队列的限制
2. **所有子队列资源总和不能超过父队列**：多个子队列的资源配额总和不能超过父队列的配额
3. **父队列必须存在**：设置父队列前，请确保父队列已创建
:::

## 创建/编辑队列

### 基础配置

| 字段 | 说明 | 限制 |
|------|------|------|
| 队列名称 | 队列唯一标识 | 创建后不可修改 |
| 父队列 | 父队列名称 | 创建后不可修改 |
| 权重 | 调度权重 | 1-65535 |
| 优先级 | 队列优先级 | 0-1000000 |
| 可回收 | 资源是否可被回收 | 开启/关闭 |

### 资源配额

| 资源 | 格式 | 示例 |
|------|------|------|
| CPU | 核数 | 8, 16, 32 |
| 内存 | Gi | 32Gi, 64Gi |
| NPU | 数量 | 8, 16, 32 |

### 常见错误

#### 队列配置冲突

当出现以下情况时会显示冲突错误：

| 错误原因 | 解决方法 |
|----------|----------|
| 子队列资源超过父队列 | 降低子队列资源配额，或增加父队列配额 |
| 所有子队列总和超过父队列 | 重新分配各子队列的资源配额 |
| 父队列不存在 | 先创建父队列，再创建子队列 |

#### 队列配置无效

| 错误原因 | 解决方法 |
|----------|----------|
| 资源格式错误 | 使用正确的格式（如 32Gi） |
| 父队列名称错误 | 确认父队列名称正确 |
| 权重/优先级超范围 | 调整到有效范围内 |

## 队列类型

| 队列 | 说明 | 适用角色 |
|------|------|----------|
| `user-{username}-compute` | 用户个人计算队列 | 学生用户专属 |
| `normal-queue` | 普通用户共享队列 | 普通用户共享 |
| `debug-sub` | 调试池子队列 | 开发环境 |

::: warning 已废弃队列
`high-queue`、`low-queue`、`debug`、`compute` 等队列已废弃。系统启动时会自动清理这些遗留队列。
:::

## 队列亲和性

计算队列（包括 `normal-queue` 和 `user-{username}-compute`）配置了节点组亲和性，确保训练任务调度到正确的节点：

| 亲和性类型 | 节点组 | 说明 |
|------------|--------|------|
| NodeGroupAffinity | `com` | 优先调度到计算节点 |
| NodeGroupAntiAffinity | `dev` | 避免调度到开发节点 |

这确保训练任务不会占用开发环境的资源，开发环境使用的 `dev` 节点组不会运行训练负载。

## 队列更新检测

系统在启动时自动检测队列配置变更，包括：

| 检测项 | 说明 |
|--------|------|
| 权重（Weight） | 调度权重变化 |
| 优先级（Priority） | 队列优先级变化 |
| 资源能力（Capability） | 资源上限变化 |
| 可回收（Reclaimable） | 资源回收属性变化 |
| 亲和性（Affinity） | 节点组亲和/反亲和变化 |

检测到变化时自动更新 Volcano Queue CRD，无需手动干预。
