# 许可管理

本文档介绍平台的许可证管理系统，包括许可证激活、状态查看和指纹绑定等功能。

## 概述

LuoSS 平台采用许可证（License）机制控制平台使用规模。许可证定义了集群可使用的最大资源限制：

| 限制项 | 说明 |
|--------|------|
| 最大节点数 | 集群允许的最大 Kubernetes 节点数 |
| 最大 NPU 数 | 集群允许的最大 NPU 总数 |
| 最大用户数 | 平台允许注册的最大用户数 |

::: warning 许可证过期
许可证过期后，平台将拒绝所有新的 API 请求（健康检查和许可证管理接口除外）。请务必在许可证过期前完成续期。
:::

## 查看许可证信息

进入 **管理后台** → **许可管理**，查看当前许可证状态。

### 许可证详情

| 字段 | 说明 |
|------|------|
| 许可证 ID | 唯一标识符 |
| 客户名称 | 授权客户名称 |
| 签发时间 | 许可证签发日期 |
| 过期时间 | 许可证有效截止日期 |
| 剩余天数 | 距离过期还剩多少天 |
| 有效状态 | 是否有效 |

### 使用情况

| 字段 | 说明 |
|------|------|
| 当前节点数 | 集群当前节点数量 |
| 当前 NPU 数 | 集群当前 NPU 总量 |
| 当前用户数 | 平台已注册用户数 |

## 激活许可证

### 获取激活码

联系平台供应商获取激活码。激活码与集群硬件指纹绑定，不可跨集群使用。

### 在线激活（推荐）

平台支持通过在线激活服务器自动验证激活码，确保每个激活码仅能使用一次。

#### 激活流程

1. 进入 **许可管理** 页面
2. 输入激活码
3. 点击 **激活**
4. 系统自动与激活服务器通信：
   - 验证激活码有效性
   - 绑定当前集群的硬件指纹
   - 标记激活码为已使用

::: info 在线/离线模式
页面会显示当前激活模式（在线/离线）。在线模式下，已使用的激活码会被拒绝（HTTP 409）。当激活服务器不可达时，系统自动回退到离线模式，直接在本地验证。
:::

### Helm 配置

```yaml
backend:
  license:
    enabled: true
    activationUrl: "https://luoss-lic.nilpo.app/"
    activationKey: "ktp-activation-xxxxxxxxx"
```

| 参数 | 说明 |
|------|------|
| `activationUrl` | 激活服务器地址，为空则仅离线模式 |
| `activationKey` | 激活服务器 API 密钥 |

### 通过 API 激活

```bash
POST /api/v1/admin/license/activate
Authorization: Bearer <admin-token>
Content-Type: application/json

{
  "activation_code": "XXXX-XXXX-XXXX-XXXX"
}
```

::: tip 硬件指纹绑定
激活码会绑定到当前集群的硬件指纹。硬件指纹基于服务器物理信息生成，重新初始化 Kubernetes 集群不会改变指纹，但更换物理服务器会导致指纹变化。
:::

## 硬件指纹

硬件指纹是基于服务器物理信息生成的唯一标识符，用于将许可证绑定到物理服务器。

### 生成方式

系统通过 Kubernetes Node API 采集每个节点的硬件标识：

1. **DMI SystemUUID**：主板的 SMBIOS/DMI 系统唯一标识符
2. **MachineID**：来自 `/etc/machine-id` 的节点机器标识

所有节点的标识按节点名排序后拼接，最终取 SHA256 哈希值作为硬件指纹。

::: info 硬件绑定优势
相比之前的 K8s CA 证书指纹，硬件指纹绑定到物理服务器而非集群证书。重新初始化 Kubernetes 集群不会改变硬件指纹，但更换物理服务器后指纹会变化。
:::

### 查看硬件指纹

```bash
GET /api/v1/admin/license/fingerprint
Authorization: Bearer <admin-token>
```

响应：

```json
{
  "hardware_fingerprint": "sha256:a1b2c3d4e5f6..."
}
```

### 指纹变更场景

| 场景 | 指纹是否变化 |
|------|-------------|
| 重新初始化 K8s 集群 | 不变 |
| 更新集群证书 | 不变 |
| 更换/添加节点 | 变化 |
| 节点主板更换 | 变化 |

## 许可证过期处理

### 过期前提醒

许可证在过期前 30 天，系统会通过以下方式提醒：

- API 响应头中添加 `X-License-Warning` 警告信息
- 管理后台显示过期提醒
- 站内信通知管理员

### 过期后影响

| 功能 | 状态 |
|------|------|
| 健康检查接口 | ✅ 正常 |
| 登录认证 | ✅ 正常 |
| 许可证管理接口 | ✅ 正常 |
| 其他所有 API | ❌ 拒绝请求 |

### 续期步骤

1. 获取当前硬件指纹
2. 将指纹发送给供应商获取新激活码
3. 在许可管理页面输入新激活码激活

## 常见问题

### Q: 如何判断许可证是否即将过期？

A: 登录管理后台，进入 **许可管理** 页面查看剩余天数。许可证在过期前 30 天会显示警告提示。

### Q: 许可证过期后数据会丢失吗？

A: 不会。许可证过期只是限制 API 访问，数据库和 Kubernetes 资源不受影响。续期后即可恢复正常使用。

### Q: 更换服务器硬件后许可证失效怎么办？

A: 硬件指纹基于服务器物理信息生成。更换服务器或添加/移除节点会导致指纹变化。需要：
1. 查看新硬件指纹
2. 联系供应商获取新激活码
3. 在许可管理页面激活

### Q: 使用量超过许可证限制会怎样？

A: 已有的资源不受影响，但新用户注册或新节点加入可能会被限制。建议在接近限制时联系供应商升级许可证。
